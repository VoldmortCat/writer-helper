# 写作技巧参考 — 供 AI 分析时使用

本文件收录常见写作技巧，供 writing_buddy.py 的 review 模式作为分析参考框架。

## 一、叙事结构

### 三幕式
- 第一幕（建置）：介绍人物、世界、冲突
- 第二幕（对抗）：矛盾升级、主角成长
- 第三幕（解决）：高潮、结局

### 英雄之旅（12 步）
平凡世界 → 冒险召唤 → 拒绝召唤 → 遇见导师 → 跨过门槛 → 考验盟友敌人 → 接近核心 → 严峻考验 → 奖赏 → 返回之路 → 复活 → 带着宝物归来

### 非线性叙事
- 倒叙（flashback）
- 多线并行（Pulp Fiction 式）
- 环形结构（结尾呼应开头）

## 二、人物塑造

### 人物维度
- 生理维度：外貌、年龄、健康状况
- 社会维度：职业、家庭、阶级
- 心理维度：欲望、恐惧、价值观

### 人物弧光（Character Arc）
- 积极弧：主角变得更好
- 消极弧：主角变得更糟
- 扁平弧：主角不变但改变世界

### 对话检查清单
- 每个角色说话方式不同吗？
- 对话推动情节或揭示性格了吗？
- 有没有"说明性对话"（两个角色说彼此已经知道的事）？

## 三、描写技巧

### 五感描写
不要只用视觉：
- 听觉：雨声、脚步、键盘声
- 嗅觉：咖啡、雨后泥土、消毒水
- 触觉：温度、质感、风
- 味觉：铁锈味、甜腻、苦涩

### 展示 vs 告诉
- 告诉：他很生气
- 展示：他攥紧拳头，指节发白，一声不吭地转身走了

### 细节选择
- 一个精准的细节胜过十句概括
- 细节要服务于人物或氛围，不要为了描写而描写

## 四、节奏控制

### 快节奏手法
- 短句、短段落
- 动作描写优先
- 省略过渡

### 慢节奏手法
- 长句、环境描写
- 内心独白
- 细节铺陈

### 节奏变化原则
- 激烈场景后接舒缓段落（让读者喘气）
- 平缓段落要埋走向高潮的线索

## 五、常见问题自查

| 问题 | 表现 | 解决 |
|------|------|------|
| 信息倾倒 | 大段背景介绍打断叙事 | 把设定拆开，在行动中展现 |
| 扁平人物 | 角色说话做事都一样 | 给每个角色一个核心欲望和恐惧 |
| 拖节奏 | 写了很多却没有推进情节 | 每一段删掉后问：读者错过什么了吗？ |
| 逻辑漏洞 | 人物行为动机不合理 | 建立人物行为规则并遵守 |
| 视角跳跃 | 一段内频繁切换 POV | 每段至少 500 字才换视角 |
| 对话标签滥用 | "他愤怒地说""她悲伤地说" | 用动作代替标签，或者干脆不用 |
| 形容词堆砌 | 每个名词前面堆三个形容词 | 删掉三分之二，留下的会更有力 |
