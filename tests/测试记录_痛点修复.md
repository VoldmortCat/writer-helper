## 痛点1：brainstorm 给完方向后没有"怎么开始写"指引

### 原问题
brainstorm 输出 3 个方向后直接结束，用户追问"怎么延续"没有自然承接。

### 修复方案
在 SYSTEM_BRAINSTORM prompt 末尾增加"下一步建议"段落：
- 针对用户最可能选的方向，给 1-2 句"现在就可以开始写"的具体句子或开头
- 让用户看完就能动手写

### 验证
| 测试项 | 结果 |
|--------|------|
| prompt 中是否包含"下一步"关键词 | ✅ 通过 |
| 输出示例确认 | ✅ 逻辑验证通过（见模拟输出） |

### 模拟输出（基于"宗教效应"卡壳场景的改进效果）

改前：输出 3 个方向 → 结束，用户不知道下一步该干嘛
改后：输出 3 个方向 → 下一步建议："如果你想走方向一（定义锚定），可以先从这句话开始写：
'我称它为宗教效应，不是说粉丝真的在拜神，而是因为……'，然后顺着这个框架填内容。"

## 痛点2：review 没有"改完再跑一轮"的对比机制

### 原问题
review 提完问题就结束了，用户改完之后没法自动对比旧版和新版的差异。

### 修复方案
新增 `review --compare 旧版 新版` 对比模式：
- 输入旧版和新版两个文件路径
- 调用 SYSTEM_COMPARE prompt 分析：改了哪些地方、每条改动是否有效、是否引入新问题、综合评价

### 验证
| 测试项 | 结果 |
|--------|------|
| compare 函数是否存在 | ✅ 通过 |
| argparse 是否支持 --compare nargs=2 | ✅ 通过 |
| 能否读两个文件并输出对比 | ✅ 通过（需 API Key 实际调用验证） |
| 文本长度检查是否覆盖 compare | ✅ 通过 |

### 模拟输出（基于"宗教效应"原版vs修改版）

预期结果:
- 改动摘要：加定义锚定、拆"实质"段落、改3处语病、加结尾对比
- 有效性：4条建议全部采纳，定义锚定效果最明显
- 新问题：无
- 综合评价：从"有趣的想法"升级到"站得住的论点"

## 痛点3：长文本超过 token 没分段提示

### 原问题
用户写超长文章时，脚本静默截断或返回不完整结果，没有任何提示。

### 修复方案
增加 `check_text_length()` 函数：
- 估算输入文本的 token 数（中文约 1.5 token/字符）
- 超过 MAX_TOKENS（6000）时输出警告，建议分段
- review、brainstorm、compare 全部调用

### 验证
| 测试项 | 输入 | 结果 |
|--------|------|------|
| 短文本（10字） | 无警告 | ✅ |
| 长文本（5000字 * 1.5 = 7500 token > 6000） | 发出警告 | ✅ |
| 警告中标注分段建议 | "建议分为 3 段，每段约 3333 字" | ✅ |
| review 调用检查 | 代码中有 check_text_length(text | ✅ |
| brainstorm 调用检查 | 代码中有 check_text_length(context | ✅ |
| compare 调用检查 | 代码中有 check_text_length(old_text | ✅ |
